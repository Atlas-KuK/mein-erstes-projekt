# Projektregeln

## Skills zuerst prüfen

Bevor du mit einer Aufgabe, Frage oder Anfrage beginnst, prüfe immer zuerst, welche Skills verfügbar sind (sichtbar im System-Reminder) und ob einer davon für die aktuelle Aufgabe sinnvoll einsetzbar ist.

Verfügbare Skills werden automatisch geladen — nutze sie proaktiv, wenn sie zur Aufgabe passen:
- `/commit` → für Git-Commits
- `/simplify` → nach Code-Änderungen für Review und Vereinfachung
- `/loop` → für wiederkehrende Aufgaben
- `claude-api` → wenn Anthropic SDK / Claude API gefragt ist
- `update-config` → für Konfigurationsänderungen an Claude Code
- `session-start-hook` → für Startup-Hooks

Wenn ein passender Skill vorhanden ist, nutze ihn — ohne dass der Benutzer explizit danach fragen muss.
